## 1. Введение

### Service

Это абстрактный объект, который определяет логический набор подов и политику доступа к ним. Сервисы создают слабую связь между подами, которые от них зависят. Сервис создаётся в формате YAML (рекомендуемый формат) или JSON, как и все остальные объекты.  
В задании мы будем создавать сервис типа `ClusterIP`.

### Gateway API

Это современный стандарт Kubernetes для управления сетевым трафиком. Он был создан как более мощная и гибкая замена старому интерфейсу Ingress.  

Основные задачи, которые решает Gateway API:
- Организация для ваших приложений внешне доступных URL  
- Обеспечение балансировки трафика  
- Терминация SSL  
- Виртуальный хостинг на основе имен и т.д.

Gateway API разделяет ответственность между ролями (админами и разработчиками):

- **GatewayClass**: Шаблон для создания шлюзов. Определяет тип балансировщика (например, «cloud-lb» или «nginx»).  
- **Gateway**: Точка входа для трафика. Здесь настраиваются IP-адреса, порты и сертификаты безопасности (TLS).  
- **HTTPRoute / GRPCRoute / TCPRoute**: Правила маршрутизации. Они указывают, какой запрос отправить в конкретный микросервис.  
- **ReferenceGrant**: Механизм безопасности, позволяющий «связывать» ресурсы из разных пространств имен (namespaces).

Компоненты Gateway API не являются стандартными объектами Kubernetes, как следствие — понадобится установить дополнительные CRD и Gateway API контроллер. В работе будем использовать Traefik.

## 4. Пошаговая инструкция выполнения домашнего задания

1. Изменить readiness-пробу в манифесте `deployment.yaml` из прошлого ДЗ на `httpGet`, вызывающую URL `/index.html`.  
2. Необходимо создать манифест `service.yaml`, описывающий сервис типа `ClusterIP`, который будет направлять трафик на поды, управляемые вашим deployment.  
3. Установить в кластер Gateway API контроллер Traefik.  
4. Создать манифест `gateway.yaml`, в котором описать объект Gateway с одним listener с протоколом HTTP и явным разрешением доступа ко всем namespace.  
5. Создать манифест `httpRoute.yaml`, в котором описать объект HTTPRoute, направляющий все HTTP-запросы к хосту `homework.otus` на ранее созданный сервис. В результате запрос `http://homework.otus/index.html` должен отдавать код HTML страницы, находящейся в подах.

### Задание с *

Доработать манифест `httpRoute.yaml`, описав в нем rewrite-правила так, чтобы обращение по адресу `http://homework.otus/homepage` форвардилось на `http://homework.otus/index.html`.
